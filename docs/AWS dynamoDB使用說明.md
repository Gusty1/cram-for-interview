## Amplify相關指令

### amplify pull

新clone下來的專案一定要先輸入這個個指令，id、key自己去csv裡面看

### amplify console

會開啟網頁進入控制台，裡面可以看到很多你的東西，這裡面最重的是API的使用期限設置，最長只能設定一年，所以要記得去調整

目前到期日期 **2027/03/14 4:00AM**

```text
API->右上角的在AppSync中檢視->左側設定->
移到中間主要授權模式->選擇API金鑰->點右上編輯->設定到期日期
```

日期設定好記得重新執行`amplify pull`

### amplify add 功能

* ​`amplify add api`​: 增加api功能，這樣才可以操作資料庫

### amplify push

簡單說就是修改模型(model)的時候記得要輸入這個指令才可以更新網路上的資料庫，更新的時候會問你要不要建立一些默認的查詢之類的，一律選是，不然之後會很麻煩

如果有刪除model的操作指令要加`--allow-destructive-graphql-schema-updates`​

### amplify update api

簡單來說，`amplify update api` 就是你的 **「API 規格修改器」**。

當你之前使用 `amplify add api` 建立好後端介面後，如果中途想要改變規則（API 金鑰期限），不需要刪除重做，用這個指令就能進入「修改模式」。

這個指令會啟動一個互動式選單，讓你調整以下核心設定：

* **修改認證方式 (Authentication)**：
* 例如：從原本簡單的 **API Key**（像你現在用的）改成更安全的 **Cognito 使用者登入**。
* 增加多種認證方式（讓某些資料只能給登入會員看，某些可以公開看）。

* **調整 API 金鑰設定**：
* 就像你剛才在網頁上改的 **過期天數**，在 CLI 裡就是透過這個指令來設定。

* **設定衝突解決策略 (Conflict Resolution)**：
* 如果你有開啟 **DataStore**（讓 App 支援離線操作），可以設定當多台裝置同時改同一筆資料時該聽誰的。

* **調整 GraphQL 效能與規則**：
* 像是自定義索引（Index）或是複雜的授權規則。

## 資料庫操作

* ​`queries.js`​: 查詢只能用這裡面的方法，可以查詢多個或單個，多個可以根據條件查詢，單個好像只能用id查詢
* ​`mutations.js`​: 新增、修改、刪除只能用這裡面的方法，可以用postman去操作，表要刪的話用push直接覆蓋過去，不然會有問題
